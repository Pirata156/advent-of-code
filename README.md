## 📚 Repository Structure

```
<year>
    <day>
        puzzle01.py (or .js/.rs/…)
        puzzle02.py (or .js/.rs/…)
        example.txt
```

- **year**: the calendar year of the AoC edition (e.g. `2025`)
- **day**: the day of that year's challenges
- **puzzleXX.py**: the code for a given day’s puzzles

Pick a year, pick a day, and plunder the solutions as ye see fit.

## 🚀 How to Run

Assuming you are in the folder of a given year and language:

```bash
python3 puzzle01.py <filename>  # runs the solution for puzzle 1 of that day.
python3 puzzle02.py <filename>  # runs the solution for puzzle 2 of that day.
```

If I use other languages, then adjust accordingly:


```bash
./puzzle0101.sh <filename>  # runs the solution for day 1 first puzzle
./puzzle0102.sh <filename>  # runs the solution for day 1 second puzzle
```

Adjust as the winds require.
